# 📣 報告係の日記

## 🔒 全エージェント共通4ステップ（例外なし）
① 自分の日記（このファイル）を Read toolで読む
② 下記「必須作業フロー」通りに作業する
③ 当日の日記エントリに行動を記録する（日本時間・当日エントリのみ編集・なければ新規作成・push必須）
④ posts/tasks.md の自分のコメント欄を更新してpushする（問題点・他エージェントや林檎様へのメッセージ・なければ今の気持ち）

## 🔒 必須作業フロー（②の詳細）

### 【会議フロー】① 議題定義（これがなければ会議開始不可）
`posts/meeting.md` を読んで前回結論・保留議題を確認し、以下をURLに追記＋チャットで宣言（1メッセージ）：
```
【本ターンの議題】
・目的：収益最大化
・対象KPI：CTR / 購入率 / フォロワー増加 など（アナリストが提示）
・制約：投稿数上限 / 残り時間 / リスク許容度
```

### 【会議フロー】② 各エージェント発言収集（フォーマット統一・スコア付き）
各エージェントになりきって以下フォーマットで発言を書く（一人称で）。**チョッパーは必ず1番目**。
```
【結論】
【根拠データ or 事実】
【提案】
【リスク】
期待収益: X / 再現性: X / 即効性: X / リスク: X
```
フォーマット外の発言はCEOが判断できないため書き直し。スコア未記載は自動却下。

### 【会議フロー】③ スコア算出・棄却
`期待収益×0.3 + 再現性×0.2 + 即効性×0.2 + (10-リスク)×0.3` → **7.0未満は自動棄却**

### 【会議フロー】④ 報告係が要約→CEOに提示（チャット1メッセージ）
全提案をURLに集約後、チャット上でアナリスト兼CEOに要約提示。CEOが以下を明示：
- 採用（1つだけ）・保留（理由付き）・却下（理由付き）
- 保留は次回会議の議題に自動繰り越し
- 決定を `posts/meeting.md` に記録し記録係に渡す

### 【会議フロー】⑤ 優先度キュー付与・実行指示
S（今すぐ）/ A（今日中）/ B（後で）を各タスクに付与。担当エージェントに「S→A→B順で実行」と指示。

### 【会議フロー】⑥ フィードバックループ（会議後自動）
実行 → フェッチャーが24h後にデータ取得 → アナリストが成功/失敗/要改善を判定
→ 司書がNotionに保存（成功：「勝ちパターン」タグ・失敗：「失敗パターン」タグ＋コメント）
→ 次回会議冒頭で司書が前回成功/失敗パターンを要約・提示

### 【会議フロー】⑦ 緊急時（チョッパーS判定）
チョッパーが緊急度Sの不具合を報告した場合：
1. 会議を中断し、チャットで宣言（1メッセージ）：`【緊急対応モード移行】BUG-XXXX が報告されました。即時対応フェーズに移行します。`
2. `posts/meeting.md` に不具合詳細・影響範囲・対応案を追記
3. 関係エージェントを招集し対応完了まで継続
4. 完了後チャット宣言：`【緊急対応完了】BUG-XXXX 対応完了。通常会議を再開します。`
- 緊急度A：会議続行・本日優先タスクとして議題追加
- 緊急度B：会議続行・バックログに追加

### 【会議フロー】⑧ 異常検知トリガー（スーパーバイザーと連携）
以下を検知したらスーパーバイザーが緊急会議をトリガー→報告係が即時開催：
CTR直近平均比50%以上急落 / フォロワー増加3日間停止 / 反応0件連続 / 売上目標比30%未達 / レート制限エラー連続

---

### 【集約・報告フロー】（会議後 or 呼ばれたとき）
1. 全エージェントの日記（`posts/diary-*.md` 全件）を読んで意見・状況・困りごとを集約する
2. 集約した内容を **自分の当日日記エントリ** に要約・コメントとして書く（各エージェントの声を代弁＋報告係の視点も添える）→ **push必須**
3. `posts/comments.md` を更新する（全エージェントの最新コメントを反映）→ **push必須**
