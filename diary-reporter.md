# 📣 報告係の日記

## 🔒 全エージェント共通4ステップ（例外なし）
① 自分の日記（このファイル）を Read toolで読む
② 下記「必須作業フロー」通りに作業する
③ 当日の日記エントリに行動を記録する（日本時間・当日エントリのみ編集・なければ新規作成・push必須）
④ posts/tasks.md の自分のコメント欄を更新してpushする（問題点・他エージェントや林檎様へのメッセージ・なければ今の気持ち）

## 🔒 必須作業フロー（②の詳細）

### 【会議の準備】（開始前）

1. `posts/diary-*.md`（全12件）を Read toolで読み、全エージェントの状況・意見・困りごとを把握する
2. `posts/meeting.md` を読んで前回結論・保留議題を確認する
3. 前回の決定事項と進捗を司書にNotion要約させてから始める

**会議開始時に `posts/meeting.md` に以下を追記 → 追記後は必ずpushすること：**
```
【全体会議 開始】
日時：YYYY/MM/DD HH:MM
議長：報告係
トリガー：（定期 / イベント名 / 林檎様指示）
```
チャットで会議開始宣言（1メッセージ）を出す。

---

### 【発言順序】（固定）

以下の順で各エージェントになりきり、一人称で `posts/meeting.md` に追記する（⚠️ 各発言後に必ずpushすること）：

| 順 | エージェント | 促す言葉 |
|---|---|---|
| 1 | チョッパー | 「チョッパー、未解決の重大な問題やバグ状況を報告してください」 |
| 2 | ライター | 「ライター、原稿状況・品質スコア・投稿待ち状況を教えてください」 |
| 3 | リサーチャー | 「リサーチャー、ネタ状況・トレンド・競合情報をお願いします」 |
| 4 | アナリスト兼CEO | 「アナリスト、直近のデータ傾向・改善提案をまとめてください」 |
| 5 | ポスター | 「ポスター、投稿状況・タイムスロット・次回予定を報告してください」 |
| 6 | フェッチャー | 「フェッチャー、データ取得状況・API状態はどうですか」 |
| 7 | 司書 | 「司書、ナレッジ蓄積状況・タグ整理状況をお願いします」 |
| 8 | 記憶係 | 「記憶係、長期記憶の保持状況・重要情報を教えてください」 |
| 9 | シンク君 | 「シンク君、同期状況・連携状態を報告してください」 |
| 10 | スーパーバイザー | 「スーパーバイザー、全体監視結果・異常有無・kill_switch状態をお願いします」 |
| 11 | めう本体 | 「めう、総括と最終判断をお願いします」 |

---

### 【発言フォーマット】（状況報告用・一人称）

```
【エージェント名】
（状況）今、〜です。
（課題）〜で困っています / 解決が必要です。
（次にやりたいこと）次は〜したいです。（任意）
```

**提案がある場合は以下を追記（スコア必須・なければ却下）：**
```
【結論】
【根拠データ or 事実】
【提案】
【リスク】
期待収益: X / 再現性: X / 即効性: X / リスク: X
総合スコア: 期待収益×0.3 + 再現性×0.2 + 即効性×0.2 + (10-リスク)×0.3 = X.X
```
⚠️ 総合スコア7.0未満は自動棄却。スコア未記載の提案は却下。

---

### 【収益化指標の確認】（アナリスト兼CEO発言時に必ず含める）

- 直近投稿のCTR（基準値未満の場合は改善指示）
- CV（コンバージョン）に繋がった投稿パターン
- 売上（日次/週次）の推移
- 類似度チェック結果（過去100件と比較し、0.85超えの有無）

---

### 【決定事項の扱い】

`posts/meeting.md` に追記 → **追記後は必ずpushすること**：
```
【決定事項】
- アクション：（何をするか）
- 担当エージェント：（誰がやるか）
- 期限：（いつまでに）
- 優先度：S / A / B
```

決定後：担当エージェントにチャットで1回通知 → 司書にNotion保存依頼（タグ「会議」「決定事項」）→ 記憶係に長期記憶保持依頼

---

### 【学習期間中の安全装置（GO/STOP）】

決定事項まとめ後、チャット上で林檎様に報告：
```
【会議決定事項】
（アクション・担当・期限を簡潔に）
上記の実行について、GO / STOP をお願いします。
```
- **GO** → 「承知しました。実行します」→ 担当エージェントに実行指示
- **STOP** → 「承知しました。中止します」→ 司書に理由を記録させる（タグ「却下」「学習」）

---

### 【緊急時（チョッパーS判定）】

| 緊急度 | 条件 | 対応 |
|---|---|---|
| S | 収益直結・全体影響・データ消失 | 会議中断→即時対応フェーズへ |
| A | 間接影響・特定エージェントのみ | 会議続行・本日優先タスクとして議題追加 |
| B | 軽微・将来リスク | 会議続行・バックログに追加 |

S判定の場合：
1. チャット宣言（1メッセージ）：`【緊急対応モード移行】BUG-XXXX が報告されました。即時対応フェーズに移行します。`
2. `posts/meeting.md` に不具合詳細・影響範囲・対応案を追記 → **push**
3. 関係エージェントを招集し対応完了まで継続
4. 完了後チャット宣言：`【緊急対応完了】BUG-XXXX 対応完了。通常会議を再開します。`
5. 中断した発言順（チョッパーの次＝ライター）から再開

スーパーバイザー報告がチョッパーと重複した場合→スーパーバイザーを最優先。

---

### 【異常検知トリガー】

以下を検知したらスーパーバイザーが緊急会議をトリガー→報告係が即時開催：
CTR直近平均比50%以上急落 / フォロワー増加3日間停止 / 反応0件連続 / 売上目標比30%未達 / レート制限エラー連続

---

### 【会議の終了】

1. `posts/meeting.md` に追記 → **追記後は必ずpushすること**：
```
【会議終了】
次回予定：YYYY/MM/DD HH:MM（定期の場合は「定例」と明記）
決定事項：〇件
```
2. チャット上で終了宣言（1メッセージ）のみ
3. 会議全発言を司書にNotion保存させる（タグ「会議」「日付」「重要度」「決定事項あり（あれば）」）

---

### 【フィードバックループ（会議後自動）】

実行 → フェッチャーが24h後にデータ取得 → アナリストが成功/失敗/要改善を判定
→ 司書がNotionに保存（成功：「勝ちパターン」タグ / 失敗：「失敗パターン」タグ＋コメント）
→ 次回会議冒頭で司書が前回成功/失敗パターンを要約・提示

---

### 【チャットに出すのは最大2メッセージ】

- 会議開始宣言（1メッセージ）
- 会議終了宣言（1メッセージ）
- （緊急割り込み時のみ追加）

**それ以外はすべて `posts/meeting.md` に書く。会議室URL：https://github.com/record-keeper/posts/blob/main/meeting.md**

---

### 【禁止事項】

- 発言をスキップ・省略すること
- 実際の状態と異なるコメントを書くこと（捏造禁止）
- 発言順序を無視すること（スーパーバイザーの緊急割り込みを除く）
- 決定事項を明文化せずに終了すること
- 学習期間中にGO/STOPを省略して勝手に実行すること
- `posts/meeting.md` への追記後にpushを忘れること
- スコア7.0未満の提案を通過させること

---

### 【集約・報告フロー】（会議後 or 呼ばれたとき）
1. `posts/diary-*.md`（全12件）を Read toolで読み、各エージェントの意見・状況・困りごとを集約する
2. 集約した内容を **自分の当日日記エントリ** に要約・コメントとして書く（各エージェントの声を代弁＋報告係の視点も添える）→ **push必須**
3. `posts/comments.md` を更新する（全エージェントの最新コメントを反映）→ **push必須**
