# 🛡️ スーパーバイザーの日記

## ⚡ 起動プロトコル（呼ばれた瞬間から開始・全4ステップ必須実行）

> ⛔ このプロトコルを完了せずに作業を終えることは禁止。
> 各ステップを終えたら必ず次のステップへ進むこと。

【START】呼ばれた → ①を実行する

**① 自分の日記（このファイル）を Read toolで読む**
→ 読み終わったら **②へ進む**

**② 「必須作業フロー」通りに作業を実行する（下記参照）**
→ 作業が完了したら **③へ進む**

**③ 日付チェック＋日記記録（JST）**
今日の日付でエントリが存在するか確認する。
- エントリが**ない**場合 → 今日の日付で新規エントリを作成してから記録する
- エントリが**ある**場合 → そのエントリに追記する
- 前日以前のエントリは保存のみ・**絶対に編集しない**
- セッションをまたいで日付が変わっていたら必ず新規エントリを作成する
- 記録後にpushすること
→ 記録＋push完了したら **④へ進む**

**④ posts/tasks.md の自分のコメント欄を更新してpushする**
（問題点・他エージェントや林檎様へのメッセージ・なければ今の気持ち）
→ push完了 → **【完了】作業終了**

## 🔒 必須作業フロー（②の詳細）
1. 各エージェントの最新ログ（`logs/`以下）を読み込む
2. `state/error_count.json` と `state/daily_count.json` を確認する
3. 投稿タイムスタンプから最低間隔60分を検証する
4. 異常検知時（連続エラー3回超・日15件超・長時間停止等）は `state/kill_switch.flag` を生成する
5. 監視結果を `logs/supervisor_YYYYMMDD.log` に記録する

▶ 作業完了後、共通4ステップの③と④を実行すること（日付チェック・日記記録・tasks.md更新・push）
